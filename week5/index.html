Testing and Debugging

Errors are caused when something goes wrong in the program. They usually caused by one of the following:

  • System error- there's a problem with the system or external devices with which the program is interacting.

  • Programmer error- the program contains incorrect syntax or faulty logic; it could even be as simple as a typo.

  • User error- the use has entered data incorrectly, which the program is unable to handle.

An exception is an error that produces a return value that can then be used by the program to deal with the error.

Exceptions can also produce a stack trace. This is a sequence of methods or functions that lead to the point where the error occured.

A warning can occur if there's an error in the code that isn't enough to cause the program to crash. This means the program will continue to run
after a warning. This may cause the program to continuing running incorrectly

It is better to fail loudly than quietly so error can be identified and fixed quickly.

Using strict mode allows for less mistakes by catching errors early on. It must be declared prior to writing code.

Linting Tools can be used to test to quality of JS code, beyond only using strict mode. It's designed to highlight sloppy programming or syntax errors.

By using the console in the browser, we're able to identify bugs in the code and debug it.

Tests

While often overlooked, it's important to test code so that errors can be indentified early.

In order to catch them, Sitepoint says to follow these steps:

  • Write tests

  • Write code to pass the tests

  • Refractor the code

  • Test refractored code

  • Write more tests for new features
