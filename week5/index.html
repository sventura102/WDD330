Debugging

Debugging is the process of a finding and fixing errors within a script.

In order to get start, you need to get to the sources panel. On windows, press f12 and go to the sources panel. It will show all the different files that made up the code on the website.

The File Navigator lists all of those things including HTML, JS, CSS, and more. Even Chrome extensions.

The Code Editor shows the source code.

The Javascript Debugging pane is for debugging.

If we press ESC then the console opens below. Here we can type commands and press ENTER to execute.

A breakpoint is a point of code where the debugger will automatically pause the JavaScript execution. When paused, we can look at the variables and try to debug it if needed.

It allows us to...

    Go to a breakpoint with the help of the right panel.
    Disable the breakpoint by unchecking it.
    Remove the breakpoint by right-clicking and selecting Remove.

There are different buttons that allow for different executions.

    The Resume key, hotkey F8, allows for the execution to continue.
    The Step key, hotkey F9, runs the next statement. Clicking it again and again will step through all the script statements one by one.
    The Step over, hotkey F10, runs the next statement but doesn't go into the function.
    The Enable/Disable button does not move the execution it just works as an on/off for breakpoints.
    The hexagon with a pause sign inside of it will cause the execution to pause when there is an error.

Coding Style

Line Length should be agreed upon at the team level. Usually 80 or 120 characters.

Horizontal Indents allows for there to be more organization so code is not so jumbled together. Easy to find things that are within other sections, classes, etc.

Function Placement is a rule that allows code, that the function is going to call, to be called first before creating the function.

Semicolons should be used after every statement.
