Week 7- Notes

Code blocks keep JS code from interpreting the wrong line. Everything within the curly brackets get called without intervening with anything outside of it. In the same note, if something outside of the curly brackets was called even though the variable is only on the inside then it will just return an error message because that code doesn't exist outside of the curly brackets. Only within it. Ex: { let message = "Hello"; (only visible in this block) alert (message): (Hello) } alert (message); (message will return in error because it's calling a variable within the code block.)

Nested Functions refers to a function created within another function. It can access any variables outside of it.

The Lexical Environment consists of two parts:

    Environmental Record stores all local variables as its properties.
    Outer Lexical Environment which is associated with any outside code.
If the variable changes throughout the code, the Lexical Environment will pick up the most recent change to the variable.

Currying is the process of breaking down a function that takes multiple arguments. It allows us to call the arguments again, allowing us to create other functions, without changing the original function.
